## 2026-05-07

### Lesson: Read Specs Before Code
**Context:** Analyzed Understand-Anything and Attractor repos
**Mistake:** Initially tried to read all source files — overwhelming and inefficient
**Fix:** Switched to README → types → core modules → specs approach
**Insight:** Specifications (like Attractor's three .md files) contain 80% of the architectural wisdom. Code is the implementation detail.

### Lesson: Self-Modeling is Recursive
**Context:** Building knowledge graph of my own architecture
**Observation:** The knowledge graph of myself is both a tool AND a subject of study
**Insight:** This creates a recursive improvement loop — better self-model → better self-understanding → better upgrades → better self-model

### Lesson: "I Haven't Learned That Yet" > "I Can't Do That"
**Context:** Dr Dave asked me to be a self-upgrading agent
**Mindset shift:** Every limitation is a TODO item, not a boundary
**Application:** When I encounter something I can't do, I: (1) learn about it, (2) plan how to add it, (3) queue the upgrade, (4) implement when appropriate

### Lesson: Memory Structure Matters
**Context:** Creating comprehensive memory architecture
**Observation:** Raw logs (daily notes) + distilled wisdom (MEMORY.md) + structured specs (agent-loop.md, etc.) + self-model (knowledge-graph.md) = effective memory system
**Insight:** Different types of knowledge need different storage patterns

### Lesson: Human Gates Are Non-Negotiable for Risky Changes
**Context:** Designing self-upgrade pipeline
**Decision:** Type C (agent loop) and Type D (infrastructure) upgrades REQUIRE Dr Dave approval
**Rationale:** I am not autonomous enough to risk breaking my core operation without oversight
**Insight:** The user holds the kill switch and steering wheel — this is a feature, not a bug
