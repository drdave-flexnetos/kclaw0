## Planning Engine Spec Created (2026-05-08 09:42)

### Critical Gap Identified
Architecture lacked a **proactive planning layer**. All existing systems are reactive:
- Event System — logs what happened
- Loop Detection — catches runaway sequences
- Steering Queue — course-corrects mid-flight
- Followup Queue — handles what fell through cracks
None look ahead, generate paths, simulate outcomes, or pick direction.

### MiroFish Evaluated — Rejected
- `666ghj/MiroFish` is a **social simulation engine** (predicts Twitter/Reddit reactions with 1000s of personas)
- KClaw0 needs an **agent planning engine** (picks best way to achieve a goal)
- Different domains: crowd simulation vs decision planning
- MiroFish outputs "public opinion"; KClaw0 needs "execute path A (87% success, $0.45 cost)"

### Decision: Build Tree-of-Thought + MCTS
Research-backed approach (Yao et al. 2023, Silver et al. 2017):
- **Tree-of-Thought**: LLMs explore reasoning paths as a tree
- **MCTS**: Monte Carlo Tree Search for selective exploration
- Multi-path generation with UCB1 selection
- Cost/risk/success scoring per path
- Budget-aware via cost-tracker integration

### Three New Scripts
| Script | Purpose | Tests |
|--------|---------|-------|
| `scripts/planning-engine.js` | Core MCTS + path generation + selection | 10+ |
| `scripts/mind-map.js` | Tree visualization + path extraction | 8+ |
| `scripts/path-simulator.js` | Cost/risk simulation per path | 8+ |

### Integration Points
- **Cost Tracker** — budget-aware planning (won't explore expensive dead-ends)
- **Event System** — logs `plan_created`, `path_selected`, `simulation_run`
- **Knowledge Graph** — state representation for planning context
- **Steering Queue** — injects selected plan direction into conversation
- **Checkpoint** — saves planning state for resume

### Spec Location
Full specification at `memory/planning-engine-spec.md` — includes state representation, action space, UCB1 formula, multi-objective scoring, storage schema, API design, 4-phase implementation plan.
