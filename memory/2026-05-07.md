### Git Pollution Incident
**Mistake:** First commit after installations included `.venv/` and `mempalace/.venv/` files (200MB+ of dependencies). GitHub push rejected due to size.

**Root cause:** `.gitignore` didn't have `.venv/` pattern. New venvs were created during installation and weren't ignored.

**Fix:**
1. Added to `.gitignore`: `.venv/`, `venv/`, `env/`, `node_modules/`
2. `git rm --cached -r` on all venv directories
3. `git reset HEAD~1` to undo bad commit
4. Re-committed clean source files only
5. Push succeeded: `f814f8e..83dc621`

**Lesson:** Always check `.gitignore` before installing dependencies in a git repo. Virtual environments must never be committed.
